# 支付宝开放平台密钥（RSA2）

## 本 Skill 解决什么

**核心**：用 **本 Skill + [reference.md](reference.md)** 中的命令与约定，在本地完成 **RSA2 密钥生成、格式说明、配置项与加验签衔接**，形成闭环；**不必**依赖支付宝桌面密钥工具，也**不必**依赖 MCP 或其它插件来「生成密钥」。

**唯一必需的外部步骤**：登录 [支付宝开放平台](https://open.alipay.com) 控制台，在应用 **开发信息** 中绑定你生成的 **应用公钥**（或证书模式下的 CSR/证书）；平台再返回 **支付宝公钥** 供你验签。这是业务规则，不是本 Skill 缺项。

## 闭环与依赖边界（必读）

| 环节 | 是否依赖本 Skill 外内容 |
|------|---------------------------|
| 生成密钥对（OpenSSL / Python） | **否**：见 reference 中命令与依赖说明 |
| 理解 PEM、环境变量、加签/验签分工 | **否**：见本 Skill 与 reference |
| 在控制台上传应用公钥、复制支付宝公钥 | **是**：必须在开放平台网页完成 |
| 调用支付宝网关、支付产品权限 | **是**：开放平台应用与产品签约，非密钥 Skill 范畴 |

**本地环境**：执行脚本前，Agent **必须**按 [reference.md「环境与依赖」](reference.md#环境与依赖) 引导用户确认 **OpenSSL 是否可用**、**Python 是否已安装所需包**（如 `cryptography`），避免直接执行后报错无说明。

## 何时使用本 Skill

- 用 **OpenSSL / Python** 在本机或 CI 生成密钥并写入项目配置时。
- 需要 **可复制命令**、**最小脚本**、或排查 **ModuleNotFoundError / openssl not found** 等与执行环境相关的问题时。
- 排查 **验签失败、公私钥不成对** 等与密钥或待签串相关的问题时。

## 本 Skill 在各环境中的加载方式

见 [reference.md「环境与工具」](reference.md#环境与工具)。

## 端到端流程（公钥模式）

1. **本地生成** RSA2（2048）密钥对：优先 **OpenSSL**（无额外 pip 依赖）；若用 **Python**，先完成 **依赖检查**（见 reference）。输出 **PKCS#8 私钥** + **SPKI 公钥** PEM。
2. **控制台**：开放平台 → 应用 → **开发信息** → **公钥模式** → 上传 **应用公钥**。
3. **保存支付宝公钥**：从控制台复制，写入 env 或文件，用于 **回调验签**。
4. **应用侧**：**应用私钥** 做请求 **RSA2 加签**；**支付宝公钥** 验签同步/异步参数。
5. **安全**：私钥不入库；`.gitignore` 排除 `*.pem`、本地 env；勿在对话中输出完整私钥。

## 证书模式（简述）

若必须 **证书模式**：用 OpenSSL 按文档生成 **CSR**，控制台完成证书与绑定；私钥与证书匹配；验签按证书或提取公钥。详见 [reference.md](reference.md)。

## 执行清单（可复制）

```text
- [ ] 已确认 OpenSSL 或 Python 环境（见 reference「环境与依赖」）
- [ ] 已生成 2048 位 RSA2 密钥对，私钥 PKCS#8、公钥可上传控制台
- [ ] 已在「开发信息」绑定应用公钥（或证书流程已完成）
- [ ] 已配置支付宝公钥与 APPID、网关（正式/沙箱）一致
- [ ] 代码中加签/验签路径已接通；密钥未进版本库
```

## Agent 行为约束

1. **优先**给出 **OpenSSL** 一条龙命令（依赖最少）；若用户指定 Python，再给出脚本，并**同步**执行 [reference.md「环境与依赖」](reference.md#环境与依赖) 中的检查与安装提示。
2. **执行或建议执行 Python 前**：提示用户确认已安装 **`cryptography`**（或其它列出的包），并给出 `python3 -m pip install cryptography`（或 `requirements.txt` 一行）及 `pip` 使用 `venv` 的推荐；遇 **ModuleNotFoundError** 按 reference 的「常见报错引导」处理。
3. **执行命令失败时**：根据 **退出码与终端原文** 对照 reference 中 **OpenSSL / openssl not found / Permission denied / PEM 解析错误** 等条目，给出下一步，不要只重复同一条命令。
4. 生成后明确三样东西：**应用私钥**、**应用公钥（上传控制台）**、**支付宝公钥（控制台下发，用于验签）**。
5. **禁止**在回复中粘贴完整私钥；验签失败时区分请求侧与回调侧原因（见 reference）。

## 环境变量映射（示例）

| 用途 | 常见 env 名 |
|------|----------------|
| APPID | `ALIPAY_APP_ID` |
| 应用私钥（加签） | `ALIPAY_APP_PRIVATE_KEY` 或 `*_PATH` |
| 支付宝公钥（验签） | `ALIPAY_PLATFORM_PUBLIC_KEY` 或 `*_PATH` |

以目标项目约定为准。
