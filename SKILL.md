# 支付宝开放平台密钥（RSA2）

## 何时使用本 Skill

在需要配置或更换 **应用密钥**（请求加签）、**支付宝公钥/证书**（回调验签）时使用。若仅排查业务参数、订单状态，应在确认密钥与加签方式无误后，再结合开放平台文档或 MCP 工具排查。

## 本 Skill 的使用方式（各环境）

本 Skill 为 **Agent Skills** 格式的说明包；在 **Cursor**、**Claude Code** 等支持该约定的工具中，将本目录放入对应 skills 目录后即可由智能体按需加载。显式调用方式、路径优先级及版本差异见 [reference.md「环境与工具」](reference.md#环境与工具)。

## 核心概念（精简）

| 角色 | 常见材料 | 存放位置 |
|------|----------|----------|
| 商户对请求签名 | **应用私钥** | 仅服务端密钥库/环境变量，禁止入仓 |
| 支付宝校验请求 | **应用公钥**（已上传）或 **应用证书** | 开放平台控制台 |
| 支付宝对回调签名 | 平台侧私钥（不可见） | — |
| 商户校验回调 | **支付宝公钥** 或 **支付宝公钥证书** | 应用配置 |

算法为开放平台约定的 **RSA2**（SHA256WithRSA）。密钥长度以控制台与官方工具默认为准，常见为 **2048** 位。

## 选择加签方式

**公钥模式**

- 使用官方密钥工具生成 **应用公钥 + 应用私钥**。
- 在控制台「开发信息」中配置 **应用公钥**，并保存控制台提供的 **支付宝公钥**，用于回调验签。

**证书模式**

- 使用官方工具按文档生成 **CSR**，在控制台完成证书申请/绑定，下载 **应用证书**、**支付宝公钥证书** 及（若文档要求）**支付宝根证书**。
- 请求签名仍使用与证书匹配的 **应用私钥**。
- 验签按 SDK/文档使用证书路径或从证书提取公钥，勿把 **CERTIFICATE** PEM 当作普通 **PUBLIC KEY** PEM 直接混用。

若产品或租户 **强制证书模式**，必须走证书流程；否则公钥模式通常更简单。

## 执行清单（可复制）

```text
- [ ] 开放平台应用已创建，「接口加签方式」与本地代码/SDK 模式一致（公钥/证书）
- [ ] 密钥材料仅通过官方「密钥工具」或文档指定流程生成（见 reference.md 文档链接）
- [ ] 已上传应用公钥或已提交 CSR 并完成证书下载
- [ ] 已从控制台复制支付宝公钥，或已保存支付宝公钥证书/根证书路径
- [ ] 应用私钥仅存在于环境变量或密钥管理，且 alipay_local.env、*.pem 已加入 .gitignore
- [ ] 联调：至少验证一笔请求加签成功 + 一条同步或异步回调验签成功
```

## Agent 行为约束

1. **生产环境**勿用临时脚本随意生成 RSA 密钥；应与官方工具导出格式一致（常见为 PKCS#8 PEM 私钥）。仅当用户明确要求本地 mock 时可例外说明风险。
2. **禁止**在对话、日志、PR 中完整输出私钥；展示时打码或仅说明已配置。
3. 映射到代码时核对加载方式：私钥多为 `BEGIN PRIVATE KEY`；验签侧若为 `BEGIN CERTIFICATE`，需按证书解析公钥后再验签。
4. 出现 **验签失败** 时，区分：**请求侧**（应用私钥与控制台应用公钥是否一对、APPID 与环境是否一致）与 **回调侧**（支付宝公钥是否当前应用最新、证书是否过期、URL 参数是否被错误解码导致待签串变化）。

## 环境变量映射（示例）

| 控制台/文件 | 常见 env 名 |
|-------------|-------------|
| APPID | `ALIPAY_APP_ID` |
| 应用私钥 | `ALIPAY_APP_PRIVATE_KEY` 或 `*_PATH` |
| 支付宝公钥（PEM 文本） | `ALIPAY_PLATFORM_PUBLIC_KEY` 或 `*_PATH` |

以目标仓库既有的命名为准。

## 轮换要点

生成新密钥 → 在控制台更新 **应用公钥/证书** → 部署新 **应用私钥** → 若控制台更新 **支付宝公钥** 则同步替换 → 分别验证下单与回调验签。
